# AGENTS.md

## Project Scope

These instructions apply to the entire `decent-development` React/Vite project.

This site is the public brochure site for DECENT Development. Keep every change aligned with the existing premium architectural direction: dark, refined, polished, construction/development focused, and consistent across the home, portfolio, team, and contact pages.

## Current Stack

- React 19 with Vite.
- React Router for routes.
- Tailwind CSS for styling.
- Framer Motion for page and section animation.
- Lucide React for icons.
- Vercel for production deployment.

## Source Of Truth

- App shell and routes: `src/App.jsx`.
- Shared page metadata: `src/hooks/usePageMeta.js`.
- Global styles and Tailwind component classes: `src/index.css`.
- Theme tokens: `tailwind.config.js`.
- Route pages: `src/pages/`.
- Shared sections/components: `src/components/`.
- Static SEO route list: `public/sitemap.xml`.
- SPA routing rewrite: `vercel.json`.

Do not patch `dist/` directly. It is generated by `npm run build`.

## Brand And Visual Direction

- Preserve the current dark architectural brand.
- Primary background: near-black `ink` / charcoal.
- Accent color: restrained gold, using existing `gold`, `gold-soft`, and bronze tokens.
- Text colors: `ivory`, `smoke`, and `stone`.
- Typography should remain premium and editorial: display headings use the configured serif display stack, body/navigation uses the configured sans stack.
- Avoid bright default SaaS styling, purple gradients, playful rounded cards, or generic template layouts.
- Keep section layouts spacious, structured, and brochure-quality.
- Use real/generated architectural imagery where pages need visual weight.
- Keep motion purposeful: subtle reveals, smooth hover depth, 3D/tilt effects where already established.
- Do not add decorative gradient orbs, random blobs, or noisy UI effects.

## Logo Rules

- The active logo asset is `src/assets/Logo.png`.
- Keep it transparent and visible on dark backgrounds.
- Do not add a white or cream box behind the logo.
- The current logo is an ivory standalone architectural mark; keep header/footer sizing compact.
- If replacing the logo, verify:
  - alpha transparency is preserved,
  - it renders on the dark header,
  - it does not rely on a background frame,
  - `npm run build` emits the expected logo asset.

## Page Structure

Keep these routes:

- `/` home
- `/completed-projects` portfolio/completed projects
- `/meet-the-team`
- `/contact`

Keep shared `Header` and `Footer` across all routes.

When adding a new route:

- Add it in `src/App.jsx`.
- Add page metadata with `usePageMeta`.
- Add or update navigation only if it improves the client-facing IA.
- Update `public/sitemap.xml`.
- Keep SPA fallback behavior in `vercel.json`.

## Copy Voice

- Use polished, premium brochure-style copy.
- Keep wording specific to residential development, duplex/triplex work, project management, and NSW construction context.
- Avoid exaggerated claims, fake guarantees, and unsupported completed-project claims.
- If using AI-generated or illustrative project images, label them as concept or illustrative imagery when necessary.
- Keep CTAs professional and direct: examples include `View portfolio`, `Start a conversation`, `Submit enquiry`, and `Get in touch`.

## Component Patterns

- Follow existing component patterns before introducing new abstractions.
- Prefer editing shared components when the change is global.
- Keep cards for repeated items only; do not wrap full page sections in card containers.
- Use existing utility classes and theme tokens rather than adding one-off colors.
- Use Lucide icons for buttons/actions when an icon is needed.
- Keep responsive behavior explicit and test desktop and mobile layouts for text overflow.

## SEO And Accessibility

- Every route should set a meaningful title, description, canonical URL, and social metadata via `usePageMeta`.
- Keep alt text descriptive and concise.
- Keep navigation labels stable and readable.
- Preserve focus states via `focus-ring`.
- Use semantic sections, headings, buttons, links, and address markup.

## Git And Deploy Workflow

Use PowerShell and explicitly set the repo location because shell working directories can drift in this environment:

```powershell
Set-Location 'C:\Users\abuba\decent\decent-development'
```

When the user asks to push or push deploy:

1. Check `git status --short`.
2. Run `npm run build`.
3. Commit the intended changes with a concise message.
4. Push `main` to `origin`.
5. Deploy production with:

```powershell
npx vercel --prod --yes
```

6. Confirm Vercel reports `READY` and the live alias includes:

```text
https://www.decentdevelopment.com.au
```

7. Smoke-check the live page or the exact affected route.

## Safety Rules

- Never use `git reset --hard` or destructive checkout commands unless the user explicitly asks.
- Do not revert unrelated user changes.
- Do not edit generated `dist/` files by hand.
- Do not expose secrets or tokens in summaries.
- Keep edits scoped to the user request.
